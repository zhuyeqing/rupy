Instructions: start {@link se.rupy.http.Daemon}, implement {@link se.rupy.http.Service} then {@link se.rupy.http.Deploy}.
Rupy is small but very potent, and it's growing stabler by the version, 0.4.2 is the first build that we can guarantee you
can safely use live without anything infront of it.

The hierarchy of the code is as follows:

   Event -+-- Query <-- Input <--- +-----------+
          |                        |           |
          X                        |  Browser  |
          |                        |           |
          +-> Reply --> Output --> +-----------+

 Where X marks the potential Comet pause point.

{@link se.rupy.http.Chain}, {@link se.rupy.http.Failure}, {@link se.rupy.http.Hash}, {@link se.rupy.http.Session} and {@link se.rupy.http.Worker} should be pretty self explanatory.
